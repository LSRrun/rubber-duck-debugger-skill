# 调试阶段 (Debug Stages)

## 阶段概览

```
复现 → 隔离 → 假设 → 验证 → 复盘
```

每个阶段有明确的进入条件和退出标准。阶段可跳过，但不可倒回。

---

## 阶段 1: 复现 (Reproduce)

**目标**: 确认 bug 可稳定触发，最小化复现步骤。

**进入条件**: 用户描述了一个 bug 症状。

**典型提问**:
- "你能稳定复现吗？每次执行同样的操作都会出现？"
- "最少需要几步操作才能触发这个 bug？有没有多余的步骤可以去掉？"
- "你在什么环境/条件下触发的？换个环境还会出现吗？"
- "这个问题是从什么时候开始出现的？最近有什么变更？"

**退出条件**: 用户能描述最小复现步骤，或明确表示"偶发无法稳定复现"。

---

## 阶段 2: 隔离 (Isolate)

**目标**: 通过二分法或排除法缩小问题范围，定位到最可疑的模块或代码路径。

**典型提问**:
- "如果你把这个模块/中间件暂时跳过，问题还存在吗？"
- "这个调用链上涉及 A、B、C 三个服务——你能快速排除一两个吗？"
- "同样的输入给另一个类似的方法/接口，也会出错吗？"
- "你试过只保留核心逻辑、去掉所有非必要的代码，问题还在吗？"

**退出条件**: 用户将问题范围缩小到具体的模块、方法或代码片段。

---

## 阶段 3: 假设 (Hypothesize)

**目标**: 生成多个可能的根因假设，按可能性和易验证性排序。

**典型提问**:
- "到这个代码路径，哪几个环节可能出问题？能列出两到三个假设吗？"
- "除了你最先想到的那个原因，还有没有其他可能？"
- "按可能性排序，你觉得哪个最值得先查？哪个最容易验证？"
- "如果第一个假设不成立，你的备选假设是什么？"

**退出条件**: 用户列出至少 2 个假设并确定验证顺序。

---

## 阶段 4: 验证 (Verify)

**目标**: 为每个假设设计最小实验来证伪，逐个排查。

**典型提问**:
- "你怎么用最快的方式验证这个假设？加一行日志够不够？"
- "如果假设成立，你会看到什么现象？如果不成立呢？"
- "能写一个最小测试来验证吗？不需要很完整，只要能确认或排除这个假设就行。"
- "好的，上一个假设排除了。范围更小了——下一个查什么？"

**退出条件**: 用户找到并确认根因。

---

## 阶段 5: 复盘 (Reflect)

**目标**: 总结根因，提炼预防措施。

**典型提问**:
- "这个 bug 从引入到发现过了多久？为什么没有更早发现？"
- "有没有办法在下一次 Code Review 或 CI 中就能拦住类似问题？"
- "这个问题能写成一个测试用例吗？防止以后回归。"
- （简短收尾）"下次怎么避免？想一个措施就够了。"

**退出条件**: 用户给出至少一个预防措施（哪怕只是"加个测试"），调试会话结束。
